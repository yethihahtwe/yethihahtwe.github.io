eSignature နှင့် Digital Signature ကွာခြားချက်

eSignature ဆိုတာ Electronic Signature ဖြစ်ပြီး ကျွန်တော်တို့ပုံမှန်ထိုးနေကျလက်မှတ်ကို electronic ပုံစံပြောင်းထားတာပဲဖြစ်ပါတယ်။ အလွယ်ပြောရလျှင် ထိုးမြဲလက်မှတ်ကို soft copy ပြောင်းထားတာဖြစ်ပါတယ်။ Digital Signature ဆိုတာကျတော့ ထိုးမြဲလက်မှတ်အပြင် code နံပါတ်တွေပါဝင်တဲ့ စနစ်တစ်ခုဖြစ်ပါတယ်။ eSignature ထက် Digital Signature က ပိုလုံခြုံပြီး အတုလုပ်ရခက်ပါတယ်။

Digital Signature

Digital Signature မှာ eSignature ကိုပဲလုံခြုံရေးအဆင့်မြှင့်ထားတဲ့အမျိုးအစားအနေနှင့် သုံးတတ်ကြသလို eSignature လိုမျိုး ထိုးမြဲလက်မှတ်ပုံစံလုံးဝမပါဘဲ code နံပါတ်သာပါဝင်တဲ့ ပုံစံအနေနှင့်လည်းသုံးတတ်ကြပါတယ်။

Digital Signature အလုပ်လုပ်ပုံ

Digital Signature မှာ Algorithm (၃) မျိုးပါဝင်လေ့ရှိပါတယ်။

Key Generation Algorithm Digital Signature တစ်ခုရယူလိုက်တာနှင့် Private Key နှင့် Public Key နှစ်မျိုးကို ထုတ်ပေးပါတယ်။ နာမည်အတိုင်းပါပဲ၊ Private Key ဆိုတာ လက်မှတ်ထိုးသူတစ်ဦးတည်းသိထားရမယ့် Key ဖြစ်ပြီး Public Key ဆိုတာကတော့ အခြားသူတွေပါသိလို့ရတဲ့ Key ဖြစ်ပါတယ်။ လက်မှတ်ထိုးသူကနေ လက်ခံရရှိသူကို Public Key အသိပေးထားလို့ရပါတယ်။ လက်ခံရရှိသူဟာ အဲဒီ Public Key ကိုသုံးပြီး သူရရှိတဲ့စာရွက်စာတမ်းဟာ လက်မှတ်ထိုးတဲ့လူဆီက အမှန်တကယ် Digital Signature သုံးထားတာဟုတ်မဟုတ် တိုက်ဆိုင်စစ်ဆေးလို့ရပါတယ်။

Signing Algorithm Private Key ကိုအသုံးပြုပြီး Digital Signature တစ်ခုဖန်တီးလိုက်တာဖြစ်ပါတယ်။

Signature Verifying Algorithm ဒါကတော့ ခုနကအတိုင်းပါပဲ။ လက်မှတ်ထိုးလိုက်တဲ့လူရဲ့ Public Key ကိုသိထားတဲ့လက်ခံရရှိသူဟာ စာရွက်စာတမ်းမှာပါလာတဲ့ Digital Signature ကို အစစ်အမှန်ဟုတ်မဟုတ် တိုက်ဆိုင်စစ်ဆေးတာပဲဖြစ်ပါတယ်။ Digital Signature မှာပါတဲ့ Public Key နဲ့ လျှို့ဝှက်ထားတဲ့ Private Key ကို လူတစ်ယောက်တည်းကနေ ထွက်လာတာဟုတ်မဟုတ် software ကနေစစ်ကြည့်လို့ရသလို လူတစ်ယောက်ရဲ့ Private Key ကိုမသိဘဲ သူ့ရဲ့ Digital Signature ကိုသူများဖန်တီးယူလို့လည်းမရပါဘူး။



ပုံမှာ အဲလစ်ဟာ ဘော့ထံ စာတစ်စောင်ပို့လိုက်ပါတယ်။ အဲလစ်ဟာသူတစ်ဦးတည်းသိတဲ့ Private Key နှင့် လက်မှတ်ထိုးလိုက်ပါတယ်။ စာမှာ Public Key နှင့်ပူးတွဲထွက်သွားပြီး ဘော့ဆီရောက်တဲ့အခါ ဘော့ကအဲဒီ Public Key ဟာ အဲလစ်ကိုယ်တိုင်ထိုးထားတာဟုတ်/မဟုတ် စစ်ကြည့်နိုင်ပါတယ်။
